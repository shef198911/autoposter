# Changelog notes

A changelog helps track user-visible changes over time.

## What to include

- new platform support
- formatting updates
- bug fixes
- command changes
- safety improvements
- documentation updates

## Suggested sections

```txt
Added
Changed
Fixed
Security
Docs
```

## Release habit

Before sharing a new archive or version, summarize the most important changes in a short changelog entry.

This makes it easier to understand which build is installed on a local machine.
