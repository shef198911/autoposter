# Versioning notes

Versioning helps track which autoposter build is being used.

## Recommended format

Use simple semantic versions when behavior changes:

```txt
1.0.0
1.1.0
1.1.1
```

## When to bump versions

- bug fixes
- formatting changes
- platform behavior changes
- new CLI commands
- new safety checks

## Documentation

When a version changes, update the release notes or changelog so it is clear what changed and why.
